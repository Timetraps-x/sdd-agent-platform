When refining brainstorming behavior, treat long-conversation context drift as a primary failure mode. The skill should preserve key decisions, compress settled context, and restate the current understanding so important facts do not get lost as the discussion grows.

**Why:** The user is concerned that long brainstorming conversations can lose critical information and that problem restatement is important for keeping alignment.

**How to apply:** In brainstorming-style interactions, add explicit checkpoints that summarize settled facts, unresolved forks, and the current problem framing before moving forward or writing artifacts.
