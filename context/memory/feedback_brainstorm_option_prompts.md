When brainstorming reaches a point where the next move depends on user choice, surface the decision explicitly and prefer selectable options.

**Why:** The user wants unresolved decision points to become concrete choices they can approve quickly, especially in real brainstorming cases where several routes are viable.

**How to apply:** In `/brainstorming`, when 2-4 valid directions remain, present them as clear options; use structured choice UI when available, and batch adjacent boundaries into one checkpoint when that reduces back-and-forth.
