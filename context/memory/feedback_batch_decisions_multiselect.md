---
name: batch decision points with multi-select
description: User prefers unresolved decision points to be consolidated so they can approve multiple boundaries at once instead of being forced into one-by-one choices.
type: feedback
---
When decisions or scope boundaries still need user approval, consolidate them into one structured checkpoint and let the user choose multiple items at once when that fits.

**Why:** The user wants to set several boundaries in one pass and does not want the interaction framed as if only one option can be chosen.

**How to apply:** When requirements or implementation trade-offs remain, group the open decision points, present them together, and use multi-select questions where appropriate instead of serial single-choice prompts.
